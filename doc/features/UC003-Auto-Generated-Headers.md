# UC-003: Headers Auto-générés

## Vue d'ensemble

Les **Headers Auto-générés** sont des en-têtes ajoutés automatiquement par l'application selon la configuration de votre requête. Cette fonctionnalité permet de visualiser ce qui sera réellement envoyé au serveur.

## Comment visualiser

1. Dans l'onglet **Headers**, localisez la section **"Auto-generated Headers"**
2. Cliquez sur **"Show"** pour déplier la liste
3. Les headers auto s'affichent avec leur **source** d'origine

![Headers auto](./assets/uc003-auto-headers.gif)

## Types de headers auto

### Headers système (toujours présents)

| Header          | Valeur              | Source |
| --------------- | ------------------- | ------ |
| User-Agent      | Rune-API-Client/1.0 | System |
| Accept          | _/_                 | System |
| Accept-Encoding | gzip, deflate, br   | System |
| Accept-Language | en-US,en;q=0.9      | System |
| Connection      | keep-alive          | System |

### Headers liés au Body

| Condition              | Header         | Valeur calculée                   |
| ---------------------- | -------------- | --------------------------------- |
| Body type = raw        | Content-Type   | Selon rawContentType              |
| Body type = formData   | Content-Type   | multipart/form-data               |
| Body type = urlEncoded | Content-Type   | application/x-www-form-urlencoded |
| Body présent           | Content-Length | Taille en bytes                   |

### Headers liés à l'Auth

| Type d'auth | Header        | Valeur         |
| ----------- | ------------- | -------------- |
| Bearer      | Authorization | Bearer {token} |
| Basic       | Authorization | Basic {base64} |

## Indication de source

Chaque header auto affiche sa source :

- **from System** : Ajouté par défaut
- **from Body** : Déduit du type de body
- **from Auth** : Injecté depuis l'onglet Auth

## Désactivation individuelle

Vous pouvez désactiver un header spécifique :

1. Trouvez le header dans la liste "Auto-generated Headers"
2. **Décochez** la case à côté du header
3. Le header grisé ne sera pas envoyé

### Cas d'usage

- Tester un serveur sans `Accept-Encoding`
- Forcer un `Content-Length` différent (débogage)
- Désactiver `User-Agent` pour anonymisation

## Différence avec les headers manuels

| Caractéristique | Headers Auto          | Headers Manuel        |
| --------------- | --------------------- | --------------------- |
| Création        | Automatique           | Par l'utilisateur     |
| Modification    | Via configuration     | Directe dans l'input  |
| Désactivation   | Checkbox dédiée       | Toggle enable/disable |
| Apparence       | Grisé/opacité réduite | Normal                |

## Fichiers techniques

- **Logique** : `src/components/request/RequestTabs.vue` (computed `autoGeneratedHeaders`)

## Cas d'erreur

| Problème                     | Solution                                    |
| ---------------------------- | ------------------------------------------- |
| Header auto ne s'affiche pas | Vérifiez la configuration (body type, auth) |
| Content-Length incorrect     | Vérifiez l'encodage du body (UTF-8)         |
